Date: July 7, 2017

The Netflix Prize: How a $1 Million Contest Changed Binge-Watching Forever

Dan Jackson

This article discusses the “why it matters.” Why do we care about recommendation systems and why would we want to build one? More specifically, it summarizes the goal of the Netflix Prize Challenge and the impact it had on computer science communities. The Challenge inspired many people from college students to experienced engineers. The excitement surrounding the Challenge, however, did not sprout from the million-dollar prize, but from the dataset itself. The Netflix Prize Challenge was before its time, in the sense that it introduced a massive dataset before “big data” was even a “thing” in the computer science world. I would like to think that the Netflix Prize Challenge is the reason that one of the most popular introductory machine learning projects is a movie recommendation system. And because of that, the Netflix Prize Challenge did not only “change binge-watching forever” but also the approach to modern-day machine learning education.

How Netflix’s Recommendations System Works

Netflix

It is important to understand the current technological complexities that are utilized in the industry. The current Netflix Recommendation System has come a long way since the Netflix Prize Challenge in 2006. The system utilizes your user data (excluding demographic data) and trends in order to make predictions about what titles you might want to watch next. The personalized Netflix recommendation system is always re-training the algorithms so that the accuracy of the predictions is constantly improving.

This Kaggle Dataset is the one that I used to create my Netflix Recommendation System. The dataset includes the 2019 titles on Netflix and includes information about the title but does not include rankings or any user information such as ratings. Date: March 27, 2019

Recommendation Systems from Scratch in Python | PYTHOLABS

Pytholabs

This article has a great introduction that explains the different types of recommendation systems and how to measure similarity. The graphics are very useful and gives the reader an overall understanding of the different methods of creating a recommendation system. The recommendation system they implement in this article also utilizes cosine similarity and is a great tutorial to follow if you were to use the exact same dataset as Pytholabs. The dataset they have contains user information and ratings which is excluded from the Netflix Dataset I found, so while this is a great example it does not directly apply to the recommendation system I implemented.

Date:January 16, 2018

Recommender Systems in Python: Beginner Tutorial

Rounak Banik

This recommender system tutorial is very helpful in helping beginners understand the basics of creating their first recommendation system. It gives a step-by-step tutorial about how to create a movie recommendation system in python and a brief summary of important concepts and terms. The concepts mentioned include content-based recommenders vs. collaborative filtering, TF-IDF, cosine similarity, countVectorizer, and further suggestions on how to improve the ecommendation system. I found this tutorial to be very informative and I utilized many of the ideas as the back-bone to my own recommendation system. I would highly suggest this tutorial to those who are just starting out because it is fairly short but allows the reader to build a rather simple, yet complete system.

Date: NaN

Cosine Similarity – Understanding the math and how it works (with python codes)

Selva Prabhakaran

This article gives an in-depth and easy-to-comprehend description of cosine-similarity. The formula that does most of the “heavy-lifting” in the recommendation system I implemented is cosine-similarity, so I believe it is very important to have a deep understanding of it. To fully understand the article, one should have some prior knowledge of linear algebra, calculus, and trigonometry. Out of many of the articles I came across when looking for more information about cosine-similarity, I liked Prabhakaran’s the best because of the graphs and very simplified examples the author includes. The three-dimensional graphs are particularly helpful to visualize (on a very small scale) what cosine-similarity calculates. Prabhakaran explains the formula in the context of finding the similarity between words in a document, which can be directly applied to the “soup” string representations of the titles that I used in my implementation of the recommendation system.
